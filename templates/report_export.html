Foolproof Operating Guide (Profit Manual)

1. Diagnosing Underperforming Models

  • Regime Drift Alert: If the Win Rate drops below 45% or average PnL turns negative, the model is likely experiencing regime shift. The market behavior has drifted from the training set. It is recommended to pause execution and clear model weights.
  • Safety Threshold Trigger: When system memory exceeds 85%, the paper trading engine automatically suspends online learning logs to prevent write locks on the local database file. Ensure the VPS or machine has free memory before resuming.
  • Optimizing Drift: Track the "Mean Absolute Drift". A value of 0.000000 indicates the optimizer is locked and is not learning. If values are too high but the win rate declines, lower the learning rate inside Settings to prevent "forgetting" generalizable patterns.

2. How to Overwrite Weights Safely for Profit

  • Trend Aligning: To force the model to buy breakouts, identify the weights of the input layer linked to index 0 (Price Trend). Making these weights slightly positive will increase long recommendation signals during upward momentum.
  • Risk Management: Scale down output weights of the Critic layer (`critic.4.weight`) to lower internal expectancy predictions, which makes the actor more conservative and less likely to hold onto losing positions.
  • Verification Rule: Any manual weight tweak triggers a 3-day verification backtest inside the Evaluation Lab. Do not execute trades with edited models until the job achieves positive net return validation.
